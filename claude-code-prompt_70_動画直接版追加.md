# Claude Code 指示: 70_プロンプト.js に動画直接版プロンプト関数を追加

## 背景

`60_worker（自動解析）.js` はすでに改修済みで、以下の5つの「動画直接版」プロンプト関数を呼んでいる。
しかし `gas/70_プロンプト.js` にはこれらの関数がまだ存在しない。追加する必要がある。

呼ばれている関数名（60_worker.jsから）:
1. `プロンプト_自分解析_単体_動画直接_()`  — 引数なし
2. `プロンプト_自分解析_テキストあり_動画直接_(userMessage)` — ユーザーメッセージあり
3. `プロンプト_比較_動画直接_(userMessage)` — プロ比較（userMessageは空文字の場合あり）
4. `プロンプト_過去比較_動画直接_(userMessage)` — 過去比較（userMessageは空文字の場合あり）
5. `プロンプト_質問モード_動画直接_(userMessage)` — 質問モード

## 重要な設計方針

### 旧方式（2段階パイプライン）と新方式（動画直接）の違い

**旧方式（text-answer経由）:**
- Stage 1: swing-analyzer が動画を見て「生テキスト（観察メモ）」を返す
- Stage 2: text-answer にプロンプト + 生テキストを渡して整形済み回答を得る
- → プロンプト関数は `基本解析`（=生テキスト）を引数に受け取り、プロンプト内に埋め込んでいた

**新方式（動画直接）:**
- swing-analyzer に動画 + プロンプトをまとめて渡し、1回のAPI呼び出しで完結
- → プロンプト関数は「基本解析テキスト」を受け取らない。Geminiが動画を直接見るので、プロンプトには「この動画を見て評価してください」という指示だけ書く
- → `【解析結果】...` のセクションは不要。代わりに「添付されたスイング動画を直接分析してください」等の指示を末尾に書く

### 質問モードの特別な設計

質問モードのプロンプトは他のモード（自分解析・プロ比較・過去比較）とは**完全に別構造**にする。
- フェーズ別評価（アドレス / テイクバック / トップ / ...）は**含めない**
- ユーザーの質問に自由形式で回答する構造にする
- ただしゴルフコーチとしての回答品質は維持する

## 追加する関数の仕様

### 追加場所
`gas/70_プロンプト.js` のファイル末尾（L1079の `プロンプト_質問モード_動画あり_` の後）に追加する。

### 関数1: `プロンプト_自分解析_単体_動画直接_()`

既存の `_プロンプト_自分解析_ベース_()` の内容をほぼそのまま使うが、末尾の「【解析結果】...」セクションを以下に置き換える:

```javascript
function プロンプト_自分解析_単体_動画直接_() {
  return [
    _プロンプト_自分解析_ベース_(),
    "",
    "---",
    "",
    "添付されたスイング動画を直接分析し、評価ルールに従って各フェーズを評価してください。",
  ].join("\n");
}
```

### 関数2: `プロンプト_自分解析_テキストあり_動画直接_(userMessage)`

既存の `_プロンプト_自分解析_補足ベース_()` を使い、ユーザーメッセージを含める:

```javascript
function プロンプト_自分解析_テキストあり_動画直接_(userMessage) {
  return [
    _プロンプト_自分解析_補足ベース_(),
    "",
    "---",
    "",
    "【ユーザーからのメッセージ】",
    '「' + String(userMessage || "").trim() + '」',
    "",
    "上記のメッセージの内容に応じて柔軟に対応してください。",
    "質問であれば回答を冒頭にまとめ、補足情報であれば考慮に入れ、注目ポイントであれば重点的に分析してください。",
    "その上で、添付されたスイング動画を直接分析し、通常の解析フォーマットで出力してください。",
  ].join("\n");
}
```

### 関数3: `プロンプト_比較_動画直接_(userMessage)`

プロ比較。userMessageが空文字かどうかで補足ベース/通常ベースを切り替える。
**重要**: 旧方式では「【プロ動画の解析】...【自分の動画の解析】...」と2つのテキストを埋め込んでいたが、動画直接版ではGeminiに動画2本が直接渡されるので、テキスト埋め込みは不要。「1本目がプロ、2本目が自分」という説明だけ書く。

```javascript
function プロンプト_比較_動画直接_(userMessage) {
  const hasMsg = userMessage && String(userMessage).trim();
  const base = hasMsg ? _プロンプト_プロ比較_補足ベース_() : _プロンプト_プロ比較_ベース_();
  const 補足行 = hasMsg ? '\n\n【ユーザーからのメッセージ】\n「' + String(userMessage).trim() + '」\n\n上記のメッセージの内容に応じて柔軟に対応してください。\n質問であれば回答を冒頭にまとめ、補足情報であれば考慮に入れ、注目ポイントであれば重点的に比較してください。\nその上で、通常の比較フォーマットで出力してください。' : "";
  return [
    base,
    "",
    "---",
    "",
    "添付された2本のスイング動画を直接分析してください。",
    "1本目がプロのお手本、2本目が私のスイングです。",
    "比較してください。",
    補足行,
  ].join("\n");
}
```

### 関数4: `プロンプト_過去比較_動画直接_(userMessage)`

過去比較。構造はプロ比較と同様。

```javascript
function プロンプト_過去比較_動画直接_(userMessage) {
  const hasMsg = userMessage && String(userMessage).trim();
  const base = hasMsg ? _プロンプト_過去比較_補足ベース_() : _プロンプト_過去比較_ベース_();
  const 補足行 = hasMsg ? '\n\n【ユーザーからのメッセージ】\n「' + String(userMessage).trim() + '」\n\n上記のメッセージの内容に応じて柔軟に対応してください。\n質問であれば回答を冒頭にまとめ、補足情報であれば考慮に入れ、注目ポイントであれば重点的に比較してください。\nその上で、通常の過去比較フォーマットで出力してください。' : "";
  return [
    base,
    "",
    "---",
    "",
    "添付された2本のスイング動画を直接分析してください。",
    "1本目が過去の私のスイング、2本目が今回の私のスイングです。",
    補足行,
  ].join("\n");
}
```

### 関数5: `プロンプト_質問モード_動画直接_(userPrompt)`

**これは他のモードとは完全に別構造にする。フェーズ別評価は含めない。**

```javascript
/** 質問モード（動画直接）：動画を見た上で、ユーザーの質問に自由形式で回答する */
function プロンプト_質問モード_動画直接_(userPrompt) {
  return [
    "あなたは世界No.1のプロのゴルフコーチです。",
    "ユーザーから添付されたスイング動画と質問が送られてきます。",
    "",
    "【回答ルール】",
    "・動画を直接見て、ユーザーの質問に丁寧かつ的確に回答する",
    "・フェーズ別評価のフォーマットは使わない。質問に対する自由形式の回答にする",
    "・動画で確認できる内容を中心にアドバイスする（見えないことは断定しない）",
    "・改善のための具体的なアドバイスやドリルがあれば積極的に教える",
    "・専門用語を使う場合は初心者にも分かるよう補足する",
    "・絵文字は適度に使ってOK",
    "",
    "【ユーザーからの質問】",
    String(userPrompt || ""),
    "",
    "添付されたスイング動画を見た上で、上記の質問に回答してください。",
  ].join("\n");
}
```

## 作業手順

1. `gas/70_プロンプト.js` を開く
2. ファイル末尾（現在のL1079 `プロンプト_質問モード_動画あり_` 関数の閉じ括弧の後）に、上記5関数を追加する
3. 追加する際、セクションヘッダーコメントをつける:
```javascript
/* =========================================================
 * 動画直接版プロンプト関数
 * swing-analyzer に動画+プロンプトを一括で渡す方式用。
 * 60_worker.js から呼ばれる。
 * text-answer（Stage 2）は使わない。
 * =======================================================*/
```
4. 既存の関数は一切変更しない（後方互換のため残す）
5. 追加後、`clasp push --force` でGASに反映
6. `clasp deploy --deploymentId AKfycbwQlDqhbTWgYrRjnMziTLkaJdF1Ja4G2PoaVS7Ubz_cdgh0HmWL24J-Flm1YgiKPRcLkQ` でデプロイ
7. `git add gas/70_プロンプト.js && git commit -m "70_プロンプト.js: 動画直接版プロンプト関数5つを追加（text-answer廃止対応）" && git push`
